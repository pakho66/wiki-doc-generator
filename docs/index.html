WorkBuddy 全功能使用说明

WorkBuddy 是腾讯出品的全场景 AI 办公工作台，核心定位是"说出要求、开始执行任务、交付完整成果"。它完美连接腾讯办公生态，是你的办公好搭子。



本文档基于 WorkBuddy 官方文档整理编写，涵盖产品所有核心功能模块的使用方法、配置指南和最佳实践，旨在帮助用户从入门到精通全面掌握 WorkBuddy。

一、产品概述

1.1 WorkBuddy 是什么

WorkBuddy 是一款由腾讯开发的 AI 驱动的全场景办公工作台。与传统的 AI 对话工具不同，WorkBuddy 不仅能进行对话交互，更重要的是它能够实际执行任务——包括读写本地文件、运行脚本、调用第三方服务、生成文档/表格/PPT、进行数据分析等。

1.2 核心能力

能力说明
理解自然语言用一句话下达任务，无需复杂的操作步骤
自主规划执行自动拆解任务、规划步骤、执行操作
多模态任务处理支持文档、表格、PPT、数据分析等多种任务类型
本地文件操作可读取授权的电脑文件夹，进行批量处理

1.3 与传统 AI 对话的区别

对比维度传统 AI 对话WorkBuddy
交互方式只能对话，提供建议能够实际执行任务
文件操作需要手动操作文件自动操作本地文件
任务复杂度单步骤简单任务多步骤复杂任务
输出形式文字回复交付可验收的结果

1.4 适用场景

  • 文档生成：生成工作报告、技术文档、会议纪要等
  • 数据分析：上传数据文件，自动分析和可视化
  • PPT/报告生成：从需求描述生成演示文稿和报告
  • 深度研究：对复杂问题进行深度调研并输出报告
  • 邮件编辑、周报生成等日常办公场景
  • 批量文件处理：自动整理、重命名、转换文件格式
  • 远程控制：通过手机即时通讯软件远程指挥电脑执行任务
  • 自动化定时任务：设置周期性重复任务的自动执行

二、安装与登录

2.1 系统要求

  • Windows：支持 Windows 10/11（暂不支持 ARM 64 架构的 Windows 11）
  • macOS：支持主流 macOS 版本
  • 安装前请确保系统已设置默认浏览器（推荐 Chrome 或 Safari）

2.2 安装步骤

  • 前往 WorkBuddy 官网（https://www.codebuddy.cn）下载对应平台的安装包
  • Windows 用户下载 .exe 安装包，双击运行即可
  • macOS 用户下载 .dmg 安装包，拖拽到应用程序文件夹即可
  • 安装完成后启动应用，会自动打开浏览器进行登录验证

2.3 登录注意事项

  • 登录时需要浏览器拉起验证窗口，请确保已设置默认浏览器
  • 如果点击登录无反应，可尝试：
  • 将 Chrome 设为默认浏览器
  • 选择"复制链接"后手动在浏览器中粘贴打开
  • 如果安全软件（火绒、360 安全卫士、电脑管家等）拦截了 WorkBuddy，请临时退出安全软件后重试
  • 如果因目录权限问题导致登录失败，需要确认应用数据目录的所有者权限是否为当前用户

2.4 账号目录结构

安装后，WorkBuddy 会在以下位置创建数据目录：

  • Windows：C:\Users\{用户名}\WorkBuddy（工作空间文件）和 C:\Users\{用户名}\AppData\Local\Programs\WorkBuddy（安装目录）
  • macOS：/Users/{用户名}/WorkBuddy（工作空间文件）

重要提示：不要在 WorkBuddy 的安装目录下存放个人项目文件，更新或重装时安装目录会被清空。个人文件应放在 WorkBuddy 工作空间目录或独立的文件夹中。


三、界面布局与核心概念

3.1 主界面结构

WorkBuddy 的主界面由以下几部分组成：

区域功能
左侧导航栏包含新建任务、项目、专家、技能、连接器、灵感、助理等功能入口
中间对话区任务对话的主要交互区域，输入指令、查看回复
右侧结果区查看任务产出的文件、代码变更和网页预览
底部输入栏任务输入框、模型选择、技能选择、连接器管理、权限切换
右上角操作栏展开/收起右侧边栏、分享、流转等操作
左下角头像区设置入口（系统设置、模型配置、数据管理、帮助与反馈等）

3.2 核心概念

  • 任务（Task）：每个对话对应一个独立任务，任务之间互不影响，各自维护独立的工作空间与上下文
  • 工作空间（Workspace）：当前任务主要读取和保存文件的文件夹
  • 产物（Artifact）：任务执行中 AI 生成的文件制品
  • 工作目录：WorkBuddy 读取和保存文件的位置，建议按任务类型分别建立目录

四、新建任务栏

新建任务栏是本地任务的统一入口，位于主界面底部或中央位置。

4.1 工作模式选择

在新建任务时，可以选择三种工作模式：

  • 问一问（Ask）：仅问答与信息查看，不修改文件
  • 做一做（Craft）：直接执行任务并修改文件
  • 想一想（Plan）：先生成执行计划，确认后再操作

4.2 切换模型

WorkBuddy 内置多个主流大模型，不同模型适用于不同场景：

模型适用场景
Auto（自动模式）根据任务自动选择适合的模型
MiniMaxExcel 处理、数据分析、PPT 生成，执行速度快
智谱 GLM多步骤、长流程的复杂任务
Kimi截图分析、图片转文档等视觉类任务
DeepSeek日常问答、文案撰写，响应快、成本低
腾讯混元中文写作、会议纪要、中文文档处理

4.3 设置工作目录

使用前建议先设置工作目录。WorkBuddy 将在该目录中读取和保存文件，未指定路径的任务也会优先在此目录内执行。

路径示例：

  • Windows：C:\Users\用户名\Desktop\工作资料
  • Mac：/Users/用户名/Desktop/工作资料

建议：按任务类型分别建立目录，例如"发票整理"、"周报"、"待处理照片"等，便于管理并降低误操作风险。

4.4 选择技能

技能（Skills）用于扩展 WorkBuddy 的任务能力。启用对应技能后，系统可完成更多特定类型的工作，如 PPT 制作、海报生成、自动化报表等。

4.5 连接第三方应用

在输入栏可快捷管理连接器，连接外部服务（QQ 邮箱、腾讯文档、腾讯会议等），扩展 AI 能力。

4.6 新建任务流程

  • 在主界面点击"新建任务"（若未提前指定目录，系统自动创建独立工作目录）
  • 输入任务内容、选择工作目录与模型后发送，即可开始执行
  • 执行中的任务显示在左侧边栏，可随时切换查看进度
  • WorkBuddy 支持多任务并行处理，可同时进行多个独立任务

五、三种工作模式

WorkBuddy 提供三种工作模式，对应不同的执行权限与交互方式，适合不同复杂度和风险级别的任务。

5.1 问一问（Ask 模式）

  • 特点：仅问答与信息查看，不修改任何文件
  • 适用场景：了解内容、确认需求、咨询问题、获取建议
  • 优势：最安全的模式，不会对系统产生任何影响
  • 推荐：首次使用优先选择 Ask 模式

5.2 做一做（Craft 模式）

  • 特点：直接执行任务并修改文件
  • 适用场景：文档生成、表格处理、文件整理等需要实际操作的任务
  • 优势：效率高，立即交付结果
  • 注意：处理重要文件前请先备份

5.3 想一想（Plan 模式）

  • 特点：先生成执行计划，确认后再操作
  • 适用场景：多步骤任务、需审阅改动范围、不确定任务影响时
  • 优势：可控性强，可在执行前审查和调整计划
  • 推荐：复杂任务或首次处理某类任务时使用

5.4 模式选择建议

场景推荐模式
第一次使用 WorkBuddyAsk
需要了解文件内容Ask
生成文档/表格/PPTCraft
整理文件、批量处理Craft（需备份）
复杂的多步骤任务Plan
不确定任务影响范围Plan
涉及重要文件修改Plan（先审计划再执行）

六、权限管理

WorkBuddy 提供两种权限模式，兼顾效率和安全。

6.1 默认权限

默认权限是日常推荐使用的模式，它像一层安全护栏：日常处理任务时不挡路，真正要越过边界或执行危险动作时，会要求你再看一眼、点一次确认。

需要确认的操作类型：

操作类型说明
写入受保护或敏感路径避免覆盖密钥、凭据或安全配置
删除受保护文件、重要目录或批量删除避免误删难以恢复的资料
执行脚本、命令或外部程序可能修改文件、访问网络或影响系统环境
网络访问或敏感能力命中安全中心规则或高风险能力时会要求确认

额外的安全保护层：

  • 沙箱执行：命令优先在沙箱约束下执行
  • 删除保护：尽量通过安全删除或回收站降低误删风险
  • 文件备份：修改已有文件前保存备份（当前仅 Windows 支持）

6.2 完全访问权限（Full Access）

完全访问权限会关闭所有二次确认。开启后，WorkBuddy 执行高风险操作时不会再逐步请求确认。

适用场景：

  • 在 Docker、虚拟机或临时测试环境中做实验
  • 在隔离目录里反复运行可信脚本

不建议开启的场景：

  • 正在处理生产资料、客户资料、财务资料或唯一副本
  • 工作目录靠近桌面、下载目录、个人文档根目录或项目仓库根目录
  • 任务会批量删除、批量重命名或覆盖文件
  • 当前电脑里有重要文件且没有备份

6.3 权限选择建议

  • 先选好工作空间：为每个任务准备独立文件夹
  • 重要文件先备份：尤其是批量处理、删除、重命名任务
  • 默认使用默认权限：让 WorkBuddy 在关键风险点向你确认
  • 看懂确认弹窗再继续：重点检查操作类型、路径和影响范围
  • 不确定就取消：让 WorkBuddy 先解释、预览或生成清单
  • 完全放开用完即关：只在可信、隔离、可恢复的环境中短时间使用

七、任务管理与对话

7.1 任务管理

任务管理面板可以查看所有任务列表，支持按状态筛选（进行中、已完成、已归档等），并继续处理已有任务。

任务特点：

  • 每个任务对应一个独立的对话和工作空间
  • 任务之间互不影响
  • 支持多任务并行处理
  • 可随时切换查看不同任务的进度

7.2 任务对话

在对话区可以持续追问、上传文件，并使用顶部操作栏的各种功能。

对话交互要点：

  • 多轮对话：对话是多轮的，第一个回答只是起点，可以不断调整和优化
  • 上传文件：可以直接拖拽文件到对话区，或通过输入栏上传
  • 追问调整：不满意结果时，直接指出问题让 WorkBuddy 修改
  • 上下文管理：如果一个会话过长出现"前说后忘"，建议开新任务

7.3 任务分享与流转

分享任务：在任务对话页顶栏点击"邀请成员"按钮，复制链接发送给他人即可。

任务流转：将任务产出物连同上下文一起移交给他人。系统会自动打包：

  • 任务产物（AI 生成的所有产出文件）
  • 任务进度摘要（WorkBuddy 自动生成的执行过程总结）
  • 自定义字段（任务标题、描述、处理人、状态、截止日期）
  • 附加附件（可额外上传补充材料）

八、结果查看

任务执行过程中，右侧会显示结果区，用来查看 WorkBuddy 产出的文件、代码变更和内置浏览器预览的内容。

8.1 四大功能标签页

标签页功能描述

十、技能（Skill）系统

10.1 什么是技能

技能（Skill）为 WorkBuddy 增加「特定工具能力」——封装一组可执行的脚本与工作流，使 AI 在授权下完成具体动作（如发送邮件、查询股价、读写文件、调用第三方 API 等）。

10.2 技能市场

技能板块分为两个区域：

  • 技能市场：推荐技能，可按需一键安装
  • 已安装：已安装的本地技能，可直接在对话中调用

10.3 如何使用技能

安装技能：

  • 点击"添加技能"
  • 三种安装方式：
  • 上传技能：导入本地技能包安装
  • 查找技能：输入任务描述，WorkBuddy 自动查找相关技能
  • 创建技能：输入任务描述，WorkBuddy 自动创建相关技能

启用/关闭技能： 已安装技能支持随时关闭或重新启用，无需卸载。

搜索技能： 技能较多时，通过搜索框输入关键词快速定位。

卸载技能： 需要完全卸载时，选择"卸载技能"，支持批量操作。

10.4 技能安全提示

  • 优先使用官方推荐 Skill
  • 安装第三方 Skill 前，仔细查看权限申请、来源说明与脚本内容
  • Skill 调用第三方、修改本地文件、执行系统命令时，均以用户本人身份执行
  • 涉及文件删除、批量写入、系统配置变更或资金类操作时，建议先小范围验证
  • 复杂任务和复杂 Skill 积分消耗通常较高

10.5 零成本精选技能

WorkBuddy 提供多个零成本 Skill，包括：

  • Agent Browser：浏览器自动化操作
  • 办公文档四件套：Word/Excel/PPT/PDF 处理
  • Local Whisper：本地语音转文字
  • yt-dlp-downloader：视频下载
  • Web Search：网络搜索
  • Obsidian：Obsidian 笔记管理
  • Skill Scanner：技能安全扫描
  • self-improvement：自我改进
  • Find Skills：查找技能
  • Frontend Design：前端设计

十一、专家中心与专家团

11.1 什么是专家

专家用于为 WorkBuddy 配置专业角色，在特定领域中能以更明确的方法和视角完成任务。

  • 专家是「角色切换」机制：以"人设 + 方法论 + 工具链"让 WorkBuddy 用特定领域专家身份执行任务
  • 专家团是「协作执行」机制：由多位专家分工协作，由团长自动拆解、并行执行并整合交付

11.2 Skill VS 专家 VS 专家团

维度Skill专家（Agent 型）专家团（Team 型）
组成关系工具能力：让 AI 能做某件事AI 顾问：懂某个领域的 AI 角色AI 协作团队：自动拆解、并行执行、完整交付
怎么选需要 AI 具备某种工具能力有一个明确的单点问题任务复杂，需要多角色配合
  • Skill 是能力
  • 专家 是能力 + 经验
  • 专家团 是多位专家 + 协作流程

11.3 如何使用

  • 打开专家中心：在左侧边栏点击"专家"，找到感兴趣的专家/专家团
  • 召唤专家/专家团：点击召唤，进入对话界面
  • 描述任务：
  • 专家：将任务告诉 WorkBuddy，会按照该角色的专业视角和方法完成任务
  • 专家团：用自然语言描述任务后，专家团团长自动拆解、分配、执行并返回完整结果

11.4 注意事项

  • 专家本身不主动获取系统权限，仅处理用户主动提供的对话与上传文件
  • 专家团会并行调用多位专家，积分消耗通常为单专家的 3-5 倍
  • WorkBuddy 生成内容仅供参考，不构成专业意见
  • 涉及医疗、法律、投资、合规等重大决策建议咨询专业人士

11.5 我的专家

在"我的专家"中可以创建属于自己的专家，分享专业知识。支持按行业分类浏览专家中心的所有专家和专家团。

十二、连接器

12.1 什么是连接器

连接器是 WorkBuddy 与外部服务之间的桥梁，用于将外部能力引入 AI 工作流，实现数据互通与能力扩展。

技术形态分两种：

  • MCP + CLI：标准化协议
  • Skill + CLI：内置脚本

12.2 支持的场景

场景说明示例
数据查询从外部数据源获取信息查询数据库、检索文档
服务调用调用第三方 API 完成操作发送邮件、创建日程
文件管理访问云端存储与文件系统读取网盘文件、上传附件
消息通知与即时通讯工具集成发送企业微信/飞书消息

12.3 已支持的连接器

  • QQ 邮箱 — 收发、搜索和整理邮件
  • 腾讯文档 — 文档操作与管理
  • 腾讯乐享 — 知识库搜索与管理
  • 腾讯会议 — 会议管理
  • TAPD — 项目管理

此外还支持自定义连接器，可接入其他外部服务。

12.4 连接器配置

入口位置：在左侧导航栏中点击"连接器"即可进入管理页面。消息框中也可快捷管理。

配置流程（以 QQ 邮箱为例）：

  • 在连接器管理页面找到目标卡片，点击"+"按钮
  • 按提示完成扫码授权或登录授权
  • 确认权限范围
  • 授权成功后，卡片名称旁出现绿色圆点

断开连接器：在已连接的连接器卡片上，点击启用/禁用开关即可。

12.5 权限与安全

  • 每个连接器独立授权
  • 不主动定时抓取数据
  • 不超出用户已有权限范围
  • 授权后可随时通过开关禁用或断开
  • 连接器自身的读写不消耗积分，WorkBuddy 对外部数据进行理解、汇总时会消耗积分

十三、项目协作

13.1 项目概述

项目是 WorkBuddy 的团队协作核心阵地，通过"项目"和"任务"两层结构，让团队成员共享上下文、标准与知识。

13.2 项目配置

配置项说明
名称项目标识
指令对 AI 的全局行为规则，所有任务自动继承
连接器项目可用的外部服务连接
专家项目可召唤的领域专家
技能（Skill）项目可调用的技能包

13.3 核心能力

1. 项目动态 — 帮助成员及时了解项目内的变化：

  • 与我有关：别人通过分享弹窗把任务定向分享或转交给我
  • 成员动态：其他成员上传/更新文件、邀请成员、公开任务等
  • 自动化：自己创建的自动化任务的通知（仅自己可见）

2. 项目任务 — 任务是项目成员在项目中创建的对话会话，一个任务对应一个对话和一个工作空间。

3. 项目资产 — 每个项目都有项目资产库，支持成员手动上传、将任务产物保存到资料库、以 RAG 形式注入任务上下文。

4. 连接器 — 分为公共授权（管理员配置、全员共用）和个人授权（各自授权、票据不共享）两种方式。

5. 自动化 — 按预设条件自动触发，当前仅支持定时执行，是个人级别的，仅创建者本人可见。

13.4 项目操作

创建项目：左侧导航栏点击"项目" → 点击"+ 新建项目" → 填写配置（也支持模板创建）

邀请成员：项目详情页右上角点击"邀请" → 复制链接发送 → 被邀请人提交申请 → 管理员审批

发起任务：在项目详情页底部输入框直接输入即可创建任务，创建时自动注入项目指令、项目资料库和个人记忆。

资产库管理：项目详情页顶部标签栏点击"资产"。每个项目默认 5GB 存储空间，支持文档、表格、幻灯片、PDF、图片、视频、音频等多种文件类型。

13.5 任务流转

任务流转是将任务产出物连同上下文一起移交给他人的功能，系统会自动打包任务产物、进度摘要、自定义字段和附加附件。接收方可基于完整的任务上下文继续推进工作。

13.6 Web 端与客户端差异

  • Web 端和客户端的个人 Skill 库互不相通
  • 项目级 Skill 在云端统一管理，双端一致
  • 云上任务中选不到客户端个人库里的 Skill，反之亦然

十四、自动化任务

14.1 什么是自动化

自动化让 WorkBuddy 按设定时间自动执行周期性、重复性任务（如每日简报、周报汇总、定时数据整理），并将结果保存到指定目录。

自动化在本地客户端保存定时任务配置，在指定时间以当前登录身份自动发起 Agent 任务。

14.2 创建自动化任务

  • 打开自动化页面，查看已安排的任务及历史执行记录
  • 点击右上角"添加"，填写配置项：
配置项说明
名称用于区分不同自动化任务
工作空间指定任务执行目录及文件保存位置
提示词描述任务目标和输出要求
选择模型和技能指定执行任务的模型和技能
定时规则设置执行频率和生效日期区间
推送到小程序开启后推送结果到 WorkBuddy 小程序

14.3 任务模板

不想从零编写提示词？可直接使用任务模板，覆盖新闻推送、周报生成、体检预约以及学习计划等常见场景，选择后按需修改即可。

14.4 推送到小程序

打开配置后，WorkBuddy 可在任务执行完成后自动将结果发送至 WorkBuddy 小程序，便于第一时间查看。

14.5 使用建议

  • 新建任务后建议先以低频率试运行，确认结果符合预期再放大调度频率
  • 对结果中涉及文件改动、数据变更、对外发送的部分，请保留可追溯的日志
  • 任务在无人值守下自动执行，请审慎设置涉及文件写入/删除/资金操作的 prompt
  • 积分消耗视任务复杂度而定，简单提醒类较低，代码巡检、文档汇总等可能产生较多 token 消耗

14.6 常见自动化场景

  • 每天 9:00 自动生成工作日报
  • 代码提交后自动跑测试
  • 每周汇总数据并生成报告
  • 定期检查代码规范
  • 定期抓取并整理行业资讯
  • 每周五下午自动归档文件

十六、记忆功能

16.1 什么是记忆

记忆让 WorkBuddy 记住你的偏好和习惯，对话越多，它就越懂你。记忆功能从会话历史中提取具有参考价值的个人信息，形成「记忆数据」并定期更新。

16.2 记忆特点

  • 提取频率：每晚整理当天会话，仅本人可见
  • 编辑记忆：可在"设置 → 记忆"中查看、编辑、删除或一键关闭
  • 默认开启：允许 WorkBuddy 从对话中提取并记住相关上下文
  • 不消耗积分：记忆提取由 WorkBuddy 自费完成

16.3 记忆管理

  • 记忆摘要每晚重新生成
  • 点击已生成的记忆，可查看、编辑记忆内容
  • 通过对话的方式告诉 WorkBuddy 要记住或忘记什么
  • 支持删除对应条目或一键关闭功能

16.4 导入记忆

支持同步在其他 AI 上的使用习惯：

  • 点击"开始导入"
  • 复制示例提示词到其他 AI 产品的对话中
  • 将结果粘贴回 WorkBuddy
  • 点击"添加到记忆"完成导入

16.5 隐私说明

  • 记忆内容遵循 WorkBuddy 隐私政策，仅本人可见
  • 会话中如涉及他人个人信息，请在提交前确认已获得相关授权
  • 关闭或删除记忆条目后，相应数据将停止用于后续对话，但已生成的输出不会追溯撤回
  • 记忆由模型自动抽取，可能存在概括偏差或时效性问题，在依赖记忆做重要判断前建议先核验

十七、设计创意（WorkBuddy × Ardot）

17.1 什么是设计创意

WorkBuddy 在「设计创意模式」中深度集成了腾讯设计 Ardot 画布——支持在对话中一句话生成 UI、PPT、海报等设计稿，实时对话修改、云端同步，可随时跳转到 Ardot 进行精细化编辑，满意后一键生成应用代码。

17.2 连接

  • 打开 WorkBuddy 首页，点击"新建任务"，选择最右侧的"设计创意"Tab
  • 首次使用时，系统会弹出授权确认窗口
  • WorkBuddy 通过登录手机号自动完成与 Ardot 的身份关联，无需单独注册

权限范围：

  • 读取画布内容：读取设计稿中的元素、样式、布局信息
  • 编辑画布：在 Ardot 画布上执行新增、修改、删除等设计操作
  • 云端同步：与 Ardot 浏览器端保持实时双向同步

17.3 使用流程

1. 对话生成设计稿：用自然语言描述你想要的设计，AI 将实时在 Ardot 画布上生成。

支持的类型：移动端 App 界面、网站页面/Landing Page、品牌 Logo、海报/Banner、PPT 演示文稿等。

2. 对话驱动修改：三种修改方式：

  • 智能框选 + 对话：选中想要调整的位置，描述修改
  • 纯语言指挥：直接描述修改需求
  • 粘贴参考：粘贴网页链接或上传参考图片

3. 跳转 Ardot 精细化编辑：对话生成帮你从 0 到 80 分，Ardot 编辑器帮你从 80 分到 100 分。

Ardot 编辑器的核心优势：

  • 所见即所得：直接拖拽、点选修改
  • 像素级精确：对齐参考线、间距标注、精确数值输入
  • 组件化编辑：复用组件、批量修改样式
  • 专业导出：PNG/SVG/PDF 多格式导出
  • 实时双向同步：两边始终一致

4. 从设计稿生成应用代码：设计稿修改满意后，可以一键生成可运行的应用代码。

17.4 推荐工作流

对话快速出稿（3 分钟）→ 跳转 Ardot 精修交付（5 分钟）→ 回到 WorkBuddy 生成代码（1 分钟）

二十二、安全与合规

22.1 三层纵深防御安全架构

1. 技术防御层 — 沙箱隔离与权限控制

  • Bash 命令沙箱：AI 操作与真实系统严格隔离
  • 文件系统隔离：只能访问预先授权的工作空间目录
  • 防沙箱逃逸：阻止对配置文件的写入
  • 默认安全设计：启动时仅拥有只读权限
  • 分层权限系统：支持 allow（允许）、ask（询问）、deny（拒绝）三种权限规则

2. 管理管控层 — 企业级统一管控

  • SSO 单点登录：统一身份认证，组织架构自动同步
  • Credit 额度控制：差异化 AI 使用额度
  • IP 白名单：限制特定 IP 段设备登录
  • 精细化工具调用权限

3. 合规审计层 — 数据保护与合规

  • Skill 安全审计：与安全团队联动，分级风险判别
  • skill-scanner 本地安全检查
  • 数据本地执行不上传
  • 服务端仅处理数据片段，用后即弃，不保存，不用于模型训练

22.2 通信安全保障

  • HTTPS 全链路加密，支持 TLS 1.2/1.3
  • IM 内置 Websocket 通道通信，不需要在防火墙开放额外端口
  • 安全通信网关，支持 Gateway Token 认证与连接鉴权

22.3 VPC 专享版安全架构

  • 物理级隔离：所有组件部署在独立 VPC 内
  • 封闭网络环境：服务间通信均在专属 VPC 内网中进行
  • 数据主权与零泄露：企业拥有绝对网络控制权和数据主权

22.4 合规资质

WorkBuddy 作为腾讯云产品，继承腾讯云安全合规体系，拥有 400+ 国内外权威认证：

二十四、10 个上手技巧

技巧 1：清晰表达，下达明确任务

记住"三要素公式"：做什么 + 有什么 + 怎么样，别让 AI 猜你的意图。

  • 反面示范：帮我把上次的会议纪要整理一下
  • 正面示范：把 D:/会议纪要/0320产品评审.docx 里的会议纪要整理成清单，包含每个议题的结论、责任人和截止日期，用表格输出

技巧 2：小步快跑，一次搞定一件事

把大任务拆成小步骤，稳稳推进。每一步先确认方向对不对，发现偏了就及时拉回来。

技巧 3：不满意就调整，别一轮定生死

对话是多轮的，第一个回答只是起点。可以直接指出问题、换个角度重述需求、补充限制条件或切换角色视角。

技巧 4：先本地后远程，稳扎稳打不踩坑

先在桌面端边看边用，摸清楚执行方式和风险边界后，再逐步放手给它远程跑。

技巧 5：给 WorkBuddy 一个身份，它就更专业

使用"专家"功能，选择合适的专家后，WorkBuddy 会优先沿着该角色的思路来组织回答。

技巧 6：给 WorkBuddy 看例子，比说一堆要求更管用

一个好的参考样本，胜过写十行抽象要求。直接把满意的样本给 WorkBuddy，并告诉它"照这个来"。

技巧 7：善用多任务，别在烂上下文里死磕

不同的任务开不同的会话，彼此互不干扰。如果一个会话过长出现"前说后忘"，直接开新任务。

技巧 8：记得频繁备份和做版本管理

让 WorkBuddy 修改文档前，最好先把原文件复制一份备份好，在文件名后面加上"_备份"或当天日期。

技巧 9：用好自动化，让 WorkBuddy 自己持续干活

通过定时任务和自动化工作流，让 WorkBuddy 在后台静默处理重复性日常工作。

技巧 10：学会"偷懒" + 合理激发 WorkBuddy 的输出

把机械性的重复工作交给 AI，把精力优先花在决策和创造性思考上。用角色设定和质量标准提升结果质量。

推荐上手顺序

  • 先学会清晰提需求
  • 再学会分步推进和多轮调整
  • 随后熟悉专家、样例和多任务能力
  • 涉及真实文件时先备份，再尝试远程和自动化

二十五、常见问题与故障排除

25.1 安装与登录问题

升级或重启后工作空间文件夹消失：

  • 在 C:/Users/用户名/WorkBuddy（Windows）或 /Users/用户名/WorkBuddy（Mac）目录中查找历史文件夹
  • 通过 WorkBuddy 打开以日期或自定义名字命名的文件夹即可找回

点击无反应，无法登录：

  • 设置 Chrome 为默认浏览器
  • 选择"复制链接"手动在浏览器中打开
  • 检查目录权限是否为当前用户
  • 临时退出安全软件后重试

25.2 文件读取问题

无法读取图片、PDF、Excel、Word 等文件：

  • 切换到支持图片或文档输入的模型
  • 分步骤描述需求
  • 安装文档处理类 Skill

生成的文件打不开：

  • 下达任务时明确说明要生成的文件格式（Word、Excel、PDF 或 Markdown）

25.3 性能问题

回复乱码或长时间无响应：

  • 切换模型测试
  • 将复杂任务拆分为更小步骤
  • 检查网络环境

任务执行卡住：

  • 点击右下角"停止任务"按钮
  • 切换到其他模型
  • 重新执行历史任务

25.4 常见错误码

错误码可能原因处理建议
14003、11133、1001模型侧状态问题切换模型重试
3002、3003、400、401、504网络环境问题检查网络，联系企业 IT
6003请求频率受限切换模型作为临时缓解

25.5 Bot 连接问题

发送消息后没有响应：

  • 确认连接状态是否在线
  • 切换模型后再次测试
  • 微信场景断开后重新连接

鸿蒙微信扫码失败：

  • 使用安卓手机完成首次扫码连接，再切换回鸿蒙微信

25.6 日志查看

  • Mac：WorkBuddy → 帮助 → 打开日志文件夹
  • Windows：WorkBuddy → 左上角帮助 → 打开日志目录

25.7 问题反馈

  • 右上角帮助 → helpFeedback → 意见反馈
  • 或右下角头像 → 设置 → 帮助与反馈 → 意见反馈
  • 描述问题，上传截图，勾选"上传日志"后提交
  • 也可发送邮件至 workbuddy@tencent.com
